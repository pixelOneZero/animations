Animations for the Web

Introduction

Implementations of web animation:

  • CSS transitions
  • CSS animations
  • SVG animations (SMIL: Snychronized Multimedia Integration Language)
  • Web Animation API
    • Status: Is in working draft form by W3C, and is partially supported by browsers
    • Model: Describes features of CSS transitions, CSS animations and SVG animations in common abstract terms.
    • Implementation: Proposes programmatic implementation of the abstract model.

CSS transitions

  • Change in a CSS property over a specified time, given a beginning and an end state.
  • List of animatable CSS properties 
  • div {transition: <property> <duration> <timing-function> <delay>;}
  • Edge cases:
    • Transitions involving property value "auto" are unpredicable, vary by browser, and are discouraged in the specification.
    • Transitions may skip to final state for elements added by appendChild() or that have just had their display:none removed. Use setTimeout() with a few milliseconds to bypass this.
CSS transition example: 
<style>
.expander {
	width:100px;
	height:50px;
	background-color:#333;
	-webkit-transition: background-color 1s, width 1s, height 1s;
	transition: background-color 1s, width 1s, height 1s;
}
.expander:hover {
	width:500px;
	height:200px;
	background-color:#aaa;
}
</style>
<div class="expander">
</div>

Web Animation API (Progress: 3.5.16 on 11/28)

  • Consists of two models:
    • Timing: a moment in time
    • Animation: contains element properties that change as a function of time.
  • The models described programmatically:
    • Player: analogous to a DVD player in its properties and methods.
      • Properties include: paused state, pending promise objects, completed state
      • Methods include: seek, pause, play, reverse play
